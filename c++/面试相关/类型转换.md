### 1. C 风格的类型转换 (C-Style Cast)

C 语言通过 `(type)expression` 的语法进行转换。

- **语法**：`(int)3.14`
- **本质**：C 风格转换是非常“粗暴”的。它会按以下顺序尝试执行四种操作之一：
  1. `const_cast`
  2. `static_cast`
  3. `static_cast` + `const_cast`
  4. `reinterpret_cast`
  5. `reinterpret_cast` + `const_cast`
- **缺点**：它无法一眼看出你的转换意图（是想做类型转换？还是想去 `const`？还是想做位操作？），且没有任何运行时或编译期的安全检查。在大型工程中，它非常难以被 `grep` 搜索到。

------

### 2. C++ 风格的类型转换 (C++ Casts)

C++ 引入了四种显式转换操作符，目的是为了让**意图明确化**，并提供相应的安全检查。

#### (1) `static_cast<new_type>(expression)`

- **场景**：用于常规的类型转换，如 `int` 到 `double`，或者父类指针到子类指针（前提是编译器能确定转换是安全的）。
- **特点**：它是编译期完成的，如果转换不合法（如不同指针类型转换），编译器会直接报错。
- **安全性**：较高。

#### (2) `dynamic_cast<new_type>(expression)`

- **场景**：专门用于**有虚函数的类层次结构**中。用于将父类指针/引用转换为子类指针/引用。
- **特点**：它在**运行时 (Runtime)** 进行检查。如果转换失败，指针返回 `nullptr`，引用则抛出 `std::bad_cast` 异常。
- **安全性**：最高，但也最慢。

#### (3) `const_cast<new_type>(expression)`

- **场景**：唯一用于**去除 `const` 或 `volatile` 属性**的工具。
- **特点**：不能改变对象的类型，只能去掉它的只读标签。
- **安全性**：如果你通过它修改了一个原本就是 `const` 的对象（比如全局常量），属于未定义行为（Undefined Behavior），极度危险。

#### (4) `reinterpret_cast<new_type>(expression)`

- **场景**：用于极其底层的操作，例如将整数转换成指针，或者将一种类型的指针转换成完全不相关的另一种指针。
- **特点**：仅仅是重新解释底层的比特位，不进行任何值的转换。
- **安全性**：最低。它打破了类型系统，极其危险，仅用于底层驱动开发或特殊的数据解析。

### C 与 C++ 转换对比表

| **特性**   | **C 风格转换**   | **C++ 风格转换**          |
| ---------- | ---------------- | ------------------------- |
| **可读性** | 低 (语法隐晦)    | 高 (明确指定意图)         |
| **安全性** | 无检查，易出错   | 有编译期/运行期检查       |
| **搜索性** | 极难 (难以 grep) | 易 (直接搜索 cast 关键字) |
| **工具箱** | 一种工具打天下   | 四种专业工具各司其职      |
