- 有符号数溢出是**未定义行为**而无符号数溢出是**定义行为**

在 C++ 中，**无符号数（Unsigned Integer）的溢出是“定义行为”**，这与有符号数溢出（这是“未定义行为”，即 UB）有着本质的区别。

简单来说，当无符号数超过了它所能表示的最大值时，它会进行**模运算（Modulo Arithmetic）**，即“回绕”（Wrap Around）。

------

**1. 它是如何工作的？（环形结构）**

你可以把无符号数想象成一个圆盘或者汽车的里程表。当里程表跑满了所有位数（比如 999999），再往前走一步，它就会归零（000000）。

对于一个 $n$ 位的无符号整数，其表示范围是 $[0, 2^n - 1]$。其溢出规律遵循以下数学公式：

$$\text{结果} = (\text{原值} + \text{加数}) \pmod{2^n}$$

**示例代码：**

```c++
#include <iostream>
#include <limits>

int main() {
    unsigned int u = std::numeric_limits<unsigned int>::max(); // 假设是 4,294,967,295
    
    std::cout << "当前值: " << u << std::endl;
    
    u = u + 1; // 溢出发生了
    
    std::cout << "加 1 之后: " << u << std::endl; // 输出 0
    
    u = u - 1; // 下溢（回绕）
    
    std::cout << "再减 1 之后: " << u << std::endl; // 输出 4294967295
    
    return 0;
}
```

------

**2. 为什么 C++ 标准将其定义为“定义行为”？**

很多加密算法、哈希算法和位操作（Bitwise Manipulation）都需要这种高效的“溢出回绕”特性。如果编译器像处理有符号数溢出那样，把无符号溢出也定为“未定义行为”并进行激进优化，那么这些依赖溢出逻辑的底层代码就会出错。

因此，C++ 标准规定：**无符号数的算术运算永远不会溢出，而是模 $2^n$ 的循环。**

------

**3. 最经典的 Bug：死循环陷阱**

理解回绕对于编写循环非常重要。这是一个新手极易犯的经典 Bug：

C++

```
// 你的意图：从 10 倒数到 0
for (unsigned int i = 10; i >= 0; i--) {
    std::cout << i << std::endl;
}
```

**为什么这是死循环？**

1. 当 `i` 变为 `0` 时，循环体执行。
2. 循环体结束后执行 `i--`。
3. `0 - 1` 在无符号数中回绕，变成了 `4,294,967,295` (即 `UINT_MAX`)。
4. `4,294,967,295 >= 0` 永远成立，循环永远不会结束。

------

**4. 对比：有符号 vs 无符号溢出**

| **比较维度**   | **有符号数 (Signed)**                      | **无符号数 (Unsigned)**  |
| -------------- | ------------------------------------------ | ------------------------ |
| **溢出行为**   | **未定义 (UB)**                            | **定义 (Modulo $2^n$)**  |
| **编译器优化** | 编译器认为永远不溢出，可能删除溢出检测代码 | 必须严格遵守回绕逻辑     |
| **安全性**     | 极其危险                                   | 可预测，但易产生逻辑错误 |
