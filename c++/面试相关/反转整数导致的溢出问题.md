在工程实践中，除了基本的翻转逻辑，**处理整数溢出（Overflow）**是这道题的重点。因为一个合法的 32 位整数（如 `2147483647`）翻转后（`7463847412`）可能会超出 32 位有符号整数的存储范围。

**C++ 实现代码**

```c++
#include <iostream>
#include <limits>

int ReverseInteger(int x) {
    int result = 0;
    
    while (x != 0) {
        // 1. 提取 x 的最后一位数字
        int pop = x % 10;
        // 2. 将 x 的最后一位截断
        x /= 10;
        
        // 3. 在将 pop 拼接到 result 之前，进行溢出安全检查
        // 检查正数溢出情况：
        // 如果 result 已经大于 INT_MAX/10，或者等于且最后一位大于 7（INT_MAX 尾数是 7）
        if (result > std::numeric_limits<int>::max() / 10 || 
           (result == std::numeric_limits<int>::max() / 10 && pop > 7)) {
            return 0; // 发生溢出，按题目常见规约返回 0（或按需抛出异常）
        }
        
        // 检查负数溢出情况：
        // 如果 result 已经小于 INT_MIN/10，或者等于且最后一位小于 -8（INT_MIN 尾数是 -8）
        if (result < std::numeric_limits<int>::min() / 10 || 
           (result == std::numeric_limits<int>::min() / 10 && pop < -8)) {
            return 0; // 发生溢出
        }
        
        // 4. 将提取的数字拼接到 result 的末尾
        result = result * 10 + pop;
    }
    
    return result;
}

int main() {
    std::cout << ReverseInteger(123) << std::endl;      // 输出: 321
    std::cout << ReverseInteger(-123) << std::endl;     // 输出: -321
    std::cout << ReverseInteger(120) << std::endl;      // 输出: 21
    std::cout << ReverseInteger(1534236469) << std::endl; // 输出: 0 (触发溢出保护)
    return 0;
}
```

**复杂度分析**

- **时间复杂度：$O(\log_{10}(|x|))$**

  循环的次数等于整数 $x$ 的十进制位数。一个整数 $x$ 的位数为 $\approx \log_{10}(|x|)$。在循环内部，每次进行的都是 $O(1)$ 常数级别的算术运算（取模、除法、比较、乘法、加法）。因此，总时间复杂度是对数级别的。

- **空间复杂度：$O(1)$**

  代码中只分配了 `result` 和 `pop` 这几个基础的整型变量，它们所占用的内存空间是恒定的，不会随着输入 $x$ 的大小而动态改变，因此空间复杂度为 $O(1)$。

**核心原理解析**

1. **负数处理：** 在 C++（C++11 及以后标准明确规定）中，对负数进行 `% 10` 取模运算，结果依然是保留符号的负数（例如 `-123 % 10 = -3`）。因此，代码内部**不需要**单独剥离负号去处理正数，同一套逻辑完美兼容正负数（`-12 * 10 + (-3) = -123`）。
2. **溢出保护机制：** 我们不能等到 `result = result * 10 + pop` 执行完之后再去判断是否溢出，因为一旦溢出，未定义行为（UB）或数值截断就已经发生了。因此，必须在乘 `10` 之前，通过判断 `result` 是否已经超过 `INT_MAX / 10` 来进行提前拦截，这是一种非常经典的防守型编程技巧。
