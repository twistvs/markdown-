- 智能指针的设计目的是为了解决new导致的内存泄露问题，为了将动态内存管理自动化，所以智能指针的使用方法和new有着极高的相似性

#### new操作符

```c++
int* p = new int(2);
```

```c++
# include <iostream>

class Person {
public:
	int a;

	Person(int a) {
		this->a = a;
	}
};

int main() {
	Person* p = new Person(2);
	std::cout << p->a << std::endl;   //输出2
	return 0;
}
```

当我们使用new操作符时，发生了以下几步操作：

1. 调用`operator new`函数，而`operator new`函数又会调用C的`malloc`，分配需要的内存
2. 初始化（调用构造函数），如果是int(2)这种内置基本数据类型，会直接把2放到刚才申请的内存；如果是类，则会调用类的相应构造函数（取决于括号内是否填了参数）
3. new操作符将这块内存的地址返回给p指针

#### std::unique_ptr

- unique_ptr会独占资源，别的unique_ptr无法同时指向这块资源（别的类型的指针更不可能，因为类型都不一样），当这个指针超过作用域或被释放时，其指向的资源也会被释放，这样既保证了资源的独占，又保证了不会出现内存溢出
- unique_ptr是一个模板类

```c++
int* p = new int(2);
//左右的int对应了上面的两个int，make_unique则对应了new，本质上功能都是一样的
std::unique_ptr<int> up1 = std::make_unique<int>(2);
//unique_ptr创建出来的是个特殊的类对象，用法和指针差不多
std::cout << *up1 << std::endl;
```

#### std::shared_ptr

- shared_ptr可以与其他shared_ptr共享资源，并且可以通过成员函数`use_count`获取指向该对象的指针的数量；当`use_count`归零时，资源自动被释放

```c++
std::shared_ptr<int> sp1 = std::make_shared<int>(2);
std::shared_ptr<int> sp2 = sp1;
//通过成员函数use_count()获取指针的个数
std::cout << sp1.use_count() << std::endl; //输出2
//通过成员函数reset()释放该指针
sp2.reset();

std::cout << sp1.use_count() << std::endl;//输出1
```

##### shared_ptr的循环引用问题

- 当两个类互相持有对方的`shared_ptr`，那么他们的`use_count`拥有无法归零，导致内存永远无法被释放

```c++
#include <iostream>
#include <memory>

struct B; // 前向声明

struct A {
    std::shared_ptr<B> ptrB;
    ~A() { std::cout << "A 被销毁了\n"; }
};

struct B {
    std::shared_ptr<A> ptrA;
    ~B() { std::cout << "B 被销毁了\n"; }
};

int main() {
    {
        auto a = std::make_shared<A>(); // a 的引用计数为 1
        auto b = std::make_shared<B>(); // b 的引用计数为 1

        a->ptrB = b; // b 的引用计数变为 2 (b 本身 + a 内部的成员)
        b->ptrA = a; // a 的引用计数变为 2 (a 本身 + b 内部的成员)
    } 
    // 退出作用域后：
    // a 销毁，a 的计数减 1 变成 1（因为 b 还指着它）
    // b 销毁，b 的计数减 1 变成 1（因为 a 还指着它）
    // 结果：由于计数不为 0，析构函数永远不会被调用！
    return 0;
}
```

#### 循环引用的解药：std::weak_ptr

- `weak_ptr`可以指向`shared_ptr`指向的对象，并且不会增加`shared_ptr`的`use_count`

```c++
# include <iostream>

class B;

class A {
public:
	int num_A;
	A(int num) :num_A(num) {}

	std::shared_ptr<B> spB;
};

class B {
public:
	int num_B;
	B(int num) : num_B(num){}
	//只需将其中一个shared_ptr改成weak_ptr即可
	std::weak_ptr<A> wpA;
};

int main() {
	std::shared_ptr<A> spA = std::make_shared<A>(1);
	std::shared_ptr<B> spB = std::make_shared<B>(2);

	spA->spB = spB;
	spB->wpA = spA;
    
    //当对象spA和spB作用域结束时，spA的计数降为0，spB降为1；然后spA->spB跟随spA一起被释放，导致spB降为0，spB的对象也被释放

	return 0;
}
```
