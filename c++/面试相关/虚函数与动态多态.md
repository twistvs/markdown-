#### 虚函数

- 首先我们知道成员函数在内存上并不属于类，而是存放在**代码区**（code segment），如果用sizeof调用一个只有成员函数的类，会输出1，也就是空类

```c++
# include <iostream>
class Person {
public:
	void func() {
		std::cout << "被调用了" << std::endl;
	}
};

int main() {
	std::cout << sizeof(Person) << std::endl;  //输出1，这个1是占位符，没有内容
	return 0;
}
```

- 如果把这个成员函数改成虚函数，sizeof会变成8，也就是一个指针

```c++
# include <iostream>
class Person {
public:
	virtual void func() {
		std::cout << "被调用了" << std::endl;
	}
};

int main() {
	std::cout << sizeof(Person) << std::endl;  //输出8，即一个指针
	return 0;
}
```

也就是说虚函数在内存上依旧不属于类，但是类会获得一个虚函数表指针vfptr，指向自己的虚函数表，而虚函数表的内容为`&Person::func`



- 当我们创建一个子类，会发现子类的sizeof也是8，这是因为子类直接把父类的vfptr复制过来了，此时子类的虚函数表和父类是完全相同的，子类可以成功调用父类的func函数

```c++
# include <iostream>
class Person {
public:
	virtual void func() {
		std::cout << "被调用了" << std::endl;
	}
};

class Son : public Person {
public:

};

int main() {
	std::cout << sizeof(Son) << std::endl;  //输出8
    Son s;
    s.func();//输出“被调用了”
	return 0;
}
```



- 如果在子类中==重写==（重写指返回值，函数名，参数列表完全相同，和重载是完全不一样的）父类的虚函数，子类会用自己的虚函数表**覆盖**父类的虚函数表，也就是`&Son::func`，此时我们创建一个子类对象，调用func，触发的是子类的func

```c++
# include <iostream>
class Person {
public:
	virtual void func() {
		std::cout << "被调用了" << std::endl;
	}
};

class Son : public Person {
public:
	void func() {
		std::cout << "子类被调用了" << std::endl;
	}
};

int main() {
	Son s;
	s.func(); //输出“子类被调用了”
	return 0;
}
```

*但是*，这并不是动态多态，用Son对象调用Son函数当然应该是这样子



- 我们多创建一个子类对象，并且用父类的指针或引用指向子类对象，会发现父类的指针或引用调用func输出的是对应的子类对象的结果，这样就实现了地址的晚绑定，编译器在运行中才会去确定func函数的真实地址

```c++
# include <iostream>
class Person {
public:
	virtual void func() {
		std::cout << "被调用了" << std::endl;
	}
};

class Son : public Person {
public:
	void func() {
		std::cout << "子类被调用了" << std::endl;
	}
};

class Son2 : public Person {
public:
	void func() {
		std::cout << "子类2被调用了" << std::endl;
	}
};

int main() {
    //创建子类对象
	Son s1;
	Son2 s2;

    //用父类的引用指向子类对象
	Person& p1 = s1;
	Person& p2 = s2;

    //调用虚函数时，调用的是子类的虚函数
	p1.func(); //输出“子类被调用了”
	p2.func(); //输出“子类2被调用了”
	
	return 0;
}
```



#### 纯虚函数与抽象类

- 在实际使用中，父类中的func往往永远都不会用到，因为父类就是当模板用的，只有子类才有实际意义；那么定义父类中的func就显得很没有意义了
- 将父类的虚函数写成纯虚函数，父类就变成了抽象类，**抽象类无法实例化**，如果子类不重写虚函数，子类也会变成抽象类
- 由于抽象类无法被实例化，所以抽象类自然无法作为函数参数和函数返回值（因为这两个都会隐式的创建一个临时的对象）

```c++
# include <iostream>
class Person {
public:
    //将父类中的虚函数写成这种形式，就变成了纯虚函数，父类就变成了抽象类
	virtual void func() = 0;
};

class Son : public Person {
public:
	void func() {
		std::cout << "子类被调用了" << std::endl;
	}
};

class Son2 : public Person {
public:
	void func() {
		std::cout << "子类2被调用了" << std::endl;
	}
};

int main() {
	Son s1;
	Son2 s2;

	Person& p1 = s1;
	Person& p2 = s2;

	p1.func();
	p2.func();
	
	return 0;
}
```



#### 虚析构与纯虚析构

- 按照以下的方式进行会导致一个问题，在删除p1指针时，父类的析构函数会被调用，但是子类的不会；如果子类new了内容到堆区，那么就不会被释放
- 将父类析构函数定义为虚函数就可以解决这个问题，子类的析构函数也会被调用

```c++
# include <iostream>
class Person {
public:
	Person() {
		std::cout << "父类的构造函数" << std::endl;
	}

	~Person() {
		std::cout << "父类的析构函数" << std::endl;
	}
};

class Son : public Person {
public:
	Son() {
		std::cout << "子类的构造函数" << std::endl;
	}

	~Son() {
		std::cout << "子类的析构函数" << std::endl;
	}
};

int main() {
	Person* p1 =  new Son;
	delete p1;
	
	return 0;
}
```

- **虚析构**一样有**纯虚析构**，但是和纯虚函数有区别：①纯虚析构*必须*在类外写函数内容（因为父类也可能有内存需要释放），②纯虚析构虽然可以在类外写内容，但是纯虚析构的父类也是抽象类（即便没有纯虚函数），无法被实例化

```c++
# include <iostream>
class Person {
public:
	Person() {
		std::cout << "父类的构造函数" << std::endl;
	}

	virtual ~Person() = 0;
		
};

//在类外写纯虚析构的内容
Person::~Person() {
	std::cout << "父类的析构函数" << std::endl;
}

class Son : public Person {
public:
	Son() {
		std::cout << "子类的构造函数" << std::endl;
	}

	~Son() {
		std::cout << "子类的析构函数" << std::endl;
	}
};

int main() {
	Person* p1 =  new Son;
	delete p1;
	
	return 0;
}
```
