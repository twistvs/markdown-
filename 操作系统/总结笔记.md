#### 自旋锁与互斥锁

**互斥锁 (Mutex)：睡眠等待**

当一个线程尝试获取互斥锁但锁已被占用时，该线程会**释放 CPU 资源**，进入“睡眠”状态。

- **挂起：** 操作系统会将该线程从调度队列中移出。
- **上下文切换：** CPU 会转去执行其他任务。
- **唤醒：** 当锁被释放时，操作系统再唤醒该线程，它重新进入就绪队列。

**自旋锁 (Spinlock)：忙等待**

当一个线程尝试获取自旋锁但锁已被占用时，该线程会**持续循环（自旋）**，不断检查锁是否可用，而不释放 CPU。

- **占着位置：** 线程始终处于运行状态，看起来像是在执行一个死循环。
- **无上下文切换：** 线程不需要进入内核态进行挂起和唤醒操作。

| **特性**              | **互斥锁 (Mutex)**                 | **自旋锁 (Spinlock)**                                       |
| --------------------- | ---------------------------------- | ----------------------------------------------------------- |
| **等待时的 CPU 占用** | 低（线程被挂起，不占 CPU）         | 高（线程在循环，占 100% CPU）                               |
| **开销**              | 高（涉及两次上下文切换的系统调用） | 低（仅需几条机器指令）                                      |
| **适合场景**          | 锁持有时间**长**的情况             | 锁持有时间**极短**的情况                                    |
| **多核要求**          | 单核、多核均可                     | **必须多核**（否则自旋会占满唯一 CPU 导致持锁线程无法运行） |

#### 分页与分段

分段会在段与段之间产生间隙，导致外部内存碎片（因为碎片在段外）

如果程序不足一页（小于4kb），系统也会给他分一整个页，导致页内有空白，导致内部内存碎片
