**判定逻辑：**

1. **`nums[mid] < nums[right]`**：
   - 说明 `mid` 到 `right` 这一段是升序的，最小值一定不在 `mid` 右侧。
   - 最小值可能是 `mid` 本身，或者在 `mid` 左侧。
   - **操作**：`right = mid`。
2. **`nums[mid] > nums[right]`**：
   - 说明旋转点（断层）就在 `mid` 到 `right` 之间。
   - 最小值一定在 `mid` 的右侧。
   - **操作**：`left = mid + 1`。

```c++
class Solution {
public:
    int findMin(vector<int>& nums) {
        int left = 0;
        int right = nums.size() - 1;

        // 注意：这里的循环条件是 left < right
        // 因为我们要找的是具体的某一个点，而不是判断 target 是否存在
        while (left < right) {
            int mid = left + (right - left) / 2;

            if (nums[mid] < nums[right]) {
                // mid 可能是最小值，所以不能写成 mid - 1
                right = mid;
            } else {
                // nums[mid] 大于右端点，说明最小值一定在右半部分
                left = mid + 1;
            }
        }
        // 最后 left == right，指向的就是最小值
        return nums[left];
    }
};
```
