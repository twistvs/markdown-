#### **初始化本地仓库**

```powershell
git init
```

------

#### **添加文件到暂存区**

```bash
git add 文件名
git add .    目录下所有文件
```

------

#### **提交到本地仓库**

```bash
git commit -m "提交说明"
```

------

#### **查看提交状态**

如果还有东西没有提交完，最好先提交

```bash
git status
```

------

#### **查看提交历史**

```bash
git log
```

------

#### **关联远程仓库**

```bash
#关联远程仓库
#每个本地仓库对于远程仓库的记录是独立的，记录在本地的.git中
git remote add origin 仓库地址
#如果你的一个本地仓库想关联几个远程仓库（这几个远程仓库可以来自不同的GitHub账号），那么每个远程仓库都得有自己的名字，不能重复
#对于本地来说，origin是这个仓库的称呼，而github自己也有一个仓库的名字，这两者并没有什么关联

#查看目前关联的远程仓库
git remote -v

#将目前commit到本地仓库的内容推送到远程仓库
git push -u origin master
#master是github的分支名字，如果不存在就会自动创建一个
#在第一次推送后，以后再推送只需要git push，origin和master已经被记住了
#如果后面想要切换别的远程仓库，就需要重新选择

git remote remove 本地对远程仓库的称呼
```

#### **云端仓库的更新**

```bash
#自动识别目录里面的新文件并且添加到暂存区
git add .

git commit -m "提交说明"

git push
```

#### 在空的文件夹中一键创建本地仓库

刚才的操作是把本地的仓库上传到github，接下来的操作是把github的内容拉取到本地

```bash
git clone 仓库地址
```

在clone的过程中，会自动生成本地.git仓库文件，也就是说，clone并不只是下载了一堆文件，而是在本地创建了一个可以和github交互的仓库

#### git的本地用户管理

github有一套自己的账户密码，git也有一套本地的用户管理方式，包含“用户名”和“邮箱“两部分，这两个东西只用于区分不同的本地用户，这些信息与github可以没有关联，但是只有邮箱与github一致的情况下，才可以在github上看到你对仓库修改的贡献条目

```bash
git config --list 
#查看本地git的本地配置
```

无论是通过终端还是通过vs，上传人的信息都只与本地配置有关，就算vs登陆的账号是twistvs，只要本地配置是twistmh，GitHub中显示的上传人就是twistmh
