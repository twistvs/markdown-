在图形渲染管线（Graphics Pipeline）中，**顶点着色器（Vertex Shader）**和**片元着色器（Fragment Shader）**是两个最基础、最重要的可编程阶段。它们分别处于管线的不同阶段，处理的数据粒度也完全不同。

简单来说：**顶点着色器决定物体“在哪”，而片元着色器决定物体“长什么样”。**

------

### 1. 顶点着色器 (Vertex Shader)

**顶点着色器是管线的第一个可编程阶段。**

- **处理单位：** 逐顶点（Per-vertex）。如果你输入一个三角形，它会运行 **3次**。
- **核心任务：** 坐标变换。将模型空间中的顶点坐标转换到裁剪空间（Clip Space）。
- **主要输入：** 顶点位置、法线、纹理坐标（UV）、切线等。
- **主要输出：** 经过变换后的坐标 `gl_Position`，以及需要传递给后续阶段的插值变量（如纹理坐标）。

在顶点着色器中，最常见的计算就是应用你之前研究过的 MVP 矩阵：



$$P_{clip} = M_{proj} \times M_{view} \times M_{model} \times P_{local}$$

------

### 2. 中间桥梁：光栅化 (Rasterization)

在顶点着色器之后，硬件会自动进行**光栅化**。

1. 它将连续的三角形切割成屏幕上一个个离散的像素点，这些点被称为**片元（Fragments）**。
2. **重心坐标插值：** 硬件利用三角形顶点的属性，根据每个片元在三角形内的相对位置，自动计算出该片元对应的属性（如颜色、UV）。

------

### 3. 片元着色器 (Fragment Shader / Pixel Shader)

**片元着色器发生在光栅化之后。**

- **处理单位：** 逐片元（Per-fragment）。如果一个三角形覆盖了屏幕上 1000 个像素，它会运行 **1000次**。
- **核心任务：** 计算颜色。根据插值后的属性、光照模型、纹理贴图等计算该像素最终的 RGBA 值。
- **主要输入：** 经过光栅化器插值后的数据（来自顶点着色器）。
- **主要输出：** 该像素的最终颜色（通常赋值给 `gl_FragColor`）。

------

### 核心差异对比

| **特性**       | **顶点着色器 (Vertex Shader)**  | **片元着色器 (Fragment Shader)**       |
| -------------- | ------------------------------- | -------------------------------------- |
| **运行频率**   | 较低（取决于模型顶点数）        | **极高**（取决于物体覆盖的屏幕像素数） |
| **关注点**     | 几何、形状、变换、位移          | 颜色、材质、光照、阴影、贴图           |
| **坐标系**     | 从模型空间到裁剪空间            | 屏幕像素空间                           |
| **必填输出**   | `gl_Position` (4D 坐标)         | `outColor` 或 `gl_FragColor` (4D 颜色) |
| **知识点联系** | 使用 **MVP 矩阵**、**仿射变换** | 使用 **重心坐标插值**、**体渲染采样**  |

------

### 形象的比喻

想象你在画一幅素描：

1. **顶点着色器**就像是你先用铅笔画出物体的**轮廓点**和结构。
2. **光栅化**就像是自动在轮廓内部打上细小的网格。
3. **片元着色器**就像是你拿着画笔，给每一个小网格**填充具体的颜色和阴影**。

### 总结

- 如果你想让一个物体**动起来**（如旗帜飘扬、水波荡漾），你应该在**顶点着色器**里修改坐标。
- 如果你想让一个物体**看起来更真实**（如金属质感、凹凸不平、发光），你应该在**片元着色器**里编写逻辑。
